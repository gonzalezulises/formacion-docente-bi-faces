## Descripción

Este módulo introductorio desarrolla el pensamiento estadístico y analítico necesario para trabajar con datos en el contexto de las ciencias sociales y la economía. Combina **conceptos fundamentales** — cómo formular preguntas analíticas, clasificar datos, reconocer sesgos y distinguir entre tipos de análisis — con el **flujo de trabajo profesional** que guía los proyectos de datos en la práctica real.

El objetivo es que cada docente pueda "pensar con datos" sobre su propia disciplina, comprender el ciclo completo de un proyecto de datos y tener un mapa claro de lo que aprenderá en los módulos siguientes.

---

## Objetivos de aprendizaje

Al completar este módulo, el participante será capaz de:

1. Distinguir entre ciencia de datos, Business Analytics, Business Intelligence, Machine Learning y estadística
2. Describir el ciclo de valor de los datos (pregunta → datos → análisis → hallazgo → acción) y aplicarlo a su disciplina
3. Comprender el flujo de trabajo profesional en ciencia de datos (Import → Tidy → Transform → Visualize → Model → Communicate)
3. Identificar las tres fases de un proyecto de datos real (Business Analytics → Machine Learning → Productivización) y ubicar su trabajo en ellas
4. Clasificar variables según su naturaleza y nivel de medición
5. Identificar sesgos comunes en el análisis de datos sociales (selección, causalidad espuria, supervivencia, sobreajuste ideológico)
6. Distinguir entre análisis descriptivo, exploratorio, inferencial, predictivo y prescriptivo
7. Analizar críticamente estudios basados en datos del ámbito social y económico

## Prerrequisitos

Ninguno. Este módulo no requiere conocimientos previos de programación ni estadística. Está diseñado para docentes de todas las carreras de FACES.

---

## Metodología

- **Conceptual, no técnico**: se prioriza la comprensión sobre la ejecución de código
- **Basado en casos**: cada concepto se ilustra con ejemplos de Economía, Administración, Contaduría y Relaciones Industriales
- **Profesionalmente contextualizado**: el flujo de trabajo y el checklist del analista conectan los conceptos con la práctica real de proyectos de datos
- **Participativo**: los notebooks incluyen ejercicios para completar durante la sesión
- **Reflexivo**: el laboratorio conecta los conceptos con la práctica docente de cada participante
